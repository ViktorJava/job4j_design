Создал класс User.
Поля класса:
int id;
String firstName;
String lastName;
int age;

Конструктор и переопределённый метод finalize().

В классе UserDemo смоделировал срабатывание GC. Для этого, 
создал два объекта класса User. Далее, переменным ссылающимся на объекты,
присвоил null, чтобы потерять ссылки на объекты и далее инициировал 
срабатывание GC. В консоли увидел удаление двух объектов:

======= Environment state =======
Free: 2
Total: 4
Max: 4
Removed: 257 Petrov
Removed: 1 Ivanov

Каждый мною созданный объект занял в памяти, как минимум 120 байт.

Заголовок объекта User 12 байт.
Пустой объект String займет 40 байт.
содержимое объектов типа String 20 байт (10 символов по 2 байта каждый)
поле типа int 4 байта.
12 + (40*2) + (4*2) + 20 = 120 байт на 1 объект типа User.

Пустой объект без полей и методов займёт в памяти:
для JVM 64 битной -  16 байт из них 12 байт заголовок и 4 байта выравнивание по 8-байтовой границе);
для JVM 32 битной - 8 байт.
